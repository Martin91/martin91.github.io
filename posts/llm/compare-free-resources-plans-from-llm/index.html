    我最近对各家 AI 之间的表现差异产生了一些兴趣，很好奇在一些热门或者尖锐话题上，AI 会不会给出语出惊人的言论，辩论是个比较适合的形式。所以我开始了 AI 辩论的项目，准备好各家大语言模型的 API 调用凭据，是我准备工作中的关键一步。

    下面是目前项目原型，后面将文本界面改成网页或者其他形式，更具趣味性！

    前面 2 个晚上，我一口气尝试了 10 个国内相对我个人来说比较知名的大语言模型，才突然发现现在各大厂商都有大量免费试用资源，对于个人体验场景根本用不完。这里整理出信息表格，方便各位感兴趣的快速了解，选择感兴趣的厂商注册试用，一起玩转 AI。

    有点特别的 LLM

    豆包

    豆包的特别之处是使用前需要手动去方舟后台模型推理页面创建推理接入点，绑定大模型之后才可以正常调用成功。

    DeepSeek

    deepseek 的注册开通流程是最低成本的，注册成功后，赠送的资源自动到账，且无需实名认证要求，其他大语言模型厂商均需要不同形式的实名认证。

    混元、百川智能

    我在 AI 辩论项目里，为了统一对接调用不同厂商大模型 API，选用了 UnionLLM 这个第三方库，它可以通过一致的接口调用国内外各大厂商的 OpenAPI。我原本以为各大厂商既然基本上都是兼容 OpenAI 家的 OpenAPI，那加上 UnionLLM 加成，接上一堆不同厂商的接口是分分钟的事，事实上我发现还是有挺多不同的。

    具体来说，我同一套代码，大多数 LLM 都是无感替换的，而唯独调用混元和百川智能，就都翻车了，原因是这两家对提示词（参数列表里的 messages）都有严格要求：

    • 至多只能有 1 个 system 消息，并且只能放在 messages 列表的第 1 个
    • 最后一个消息必须是 user 消息或者 tool 消息

    另外就是 UnionLLM 其实并没有官方支持混元，而我是通过它依赖的 LiteLLM 库实现了绕行，混元加油，目前从各大榜单和开发者体验，给我的感觉有点差强人意。

    讯飞星火、百度文心一言

    我原本对这两家还挺期待的，但是纳闷的是，这两家的短信验证码我一直收不到，反复尝试几次之后，我就放弃了，后面还会再试试，不过尝试的第一步就遇挫，还挺让人失望的。
